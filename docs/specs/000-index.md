# Specs Index

Use this directory for durable product and technical specifications.

## Rules

- Create or link a GitHub issue before active work starts
- Add `Linked issue: #...` near the top of each spec
- Keep status current: Draft, In Review, Approved, Archived
- Link follow-up implementation issues where relevant

## Suggested workflow

1. Open a spec issue
2. Draft spec from `docs/templates/spec-template.md`
3. Review and refine in PRs/issues
4. Mark approved and split delivery into feature issues if needed
