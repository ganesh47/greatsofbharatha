# greatsofbharatha

Product repo for Greats of Bharatha.

## Working model

- Product planning lives in GitHub Issues
- Delivery is tracked in GitHub Projects
- Roadmap slices live in Milestones
- Research is documented in `docs/research/`
- Specs are documented in `docs/specs/`
- Active work should link docs and issues together

## Repo structure

- `docs/specs/` - feature and system specs
- `docs/research/` - research notes and investigations
- `.github/` - issue templates, workflows, automation

## Conventions

- Every meaningful work item gets an issue
- Every active spec/research document should reference its related issue
- Milestones represent phases or releases, not one-off tasks
