# Project Setup and Working Conventions

## Tracker project

Project title: `Greats of Bharatha Tracker`

Recommended fields in GitHub Project:
- Status
- Priority
- Area
- Size
- Target milestone

Recommended status flow:
- Backlog
- Ready
- In Progress
- Blocked
- Done

## Repo conventions

- Specs live in `docs/specs/`
- Research lives in `docs/research/`
- Active work should be tracked in Issues
- Issues should carry a status label and, when relevant, a priority label
- Milestones should group phases/releases, not micro-tasks

## Labels

Core labels:
- `spec`
- `research`
- `feature`
- `bug`
- `documentation`

Priority labels:
- `priority/critical`
- `priority/high`
- `priority/medium`
- `priority/low`

Status labels:
- `status/backlog`
- `status/active`
- `status/blocked`
- `status/stale`

## Recommended issue lifecycle

1. Create issue from template
2. Assign milestone and labels
3. Add or link spec/research doc
4. Track discussion and decisions in issue/PRs
5. Close with linked PR/doc updates
